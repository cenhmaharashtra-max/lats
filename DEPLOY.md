## Option A: Railway (Recommended — Easiest, Free Tier)

### Step 1: Create accounts
1. Sign up at https://railway.app (free)
2. Sign up at https://github.com (free) if you don't have one

### Step 2: Push code to GitHub
1. Download and unzip this deployment package
2. Open terminal / command prompt in the folder
3. Run these commands:
```
git init
git add .
git commit -m "LATS initial deployment"
```
4. Create a new repository at https://github.com/new (name it "lats")
5. Run:
```
git remote add origin https://github.com/YOUR_USERNAME/lats.git
git push -u origin main
```

### Step 3: Deploy on Railway
1. Go to https://railway.app → New Project
2. Click **"Deploy from GitHub repo"** → select your "lats" repo
3. Railway will auto-detect Node.js and start deploying

### Step 4: Add PostgreSQL database
1. In your Railway project → click **"+ New"** → **"Database"** → **"PostgreSQL"**
2. Click the Postgres service → **"Variables"** tab
3. Copy the `DATABASE_URL` value

### Step 5: Set environment variables
1. Click your LATS app service → **"Variables"** tab
2. Add these variables:
   - `DATABASE_URL` = paste the value from Step 4
   - `SESSION_SECRET` = any random string (e.g. `lats-prod-secret-2024-xyz`)
   - `NODE_ENV` = `production`
3. Click **"Deploy"**

### Step 6: Access your app
- Railway gives you a URL like: `https://lats-production.up.railway.app`
- Open it and login with CENH / CENHMAH123

## Option B: Render (Also Free)

1. Sign up at https://render.com
2. Push code to GitHub (same as Step 2 above)
3. New → Web Service → connect GitHub repo
4. Settings:
   - Build Command: `npm install`
   - Start Command: `npm start`
5. New → PostgreSQL → create database
6. Copy Internal Database URL
7. In your web service → Environment:
   - `DATABASE_URL` = paste database URL
   - `SESSION_SECRET` = random string
   - `NODE_ENV` = `production`
8. Deploy

## Option C: VPS / Local Server

### Prerequisites
- Node.js 18+ installed
- PostgreSQL installed and running

### Setup
```bash
# 1. Unzip deployment package
cd lats-deploy

# 2. Install dependencies
npm install

# 3. Create .env file
cp .env.example .env
# Edit .env with your actual DATABASE_URL and SESSION_SECRET

# 4. Start server
npm start
```

### Run with PM2 (keeps app running after reboot)
```bash
npm install -g pm2
pm2 start server.js --name lats
pm2 startup   # auto-start on reboot
pm2 save
```

## Troubleshooting

- **"Cannot connect to database"** → Check DATABASE_URL in environment variables
- **"Session error"** → Set SESSION_SECRET environment variable
- **Blank page** → Check app logs in Railway/Render dashboard
- **Login fails** → Database may not have initialized — restart the service
